## 3. UI — Estructura del Sistema

### 3.1 Arquitectura General

```
app.blade.php  (Layout principal)
├── Sidebar (aside)
│   ├── Botón colapsar
│   ├── Logo + nombre
│   └── Navegación con permisos
├── Header
│   ├── Título del módulo (slot)
│   ├── Breadcrumb "Panel de gestión operativa"
│   └── Dropdown perfil (config, ayuda, cerrar sesión)
└── Main content ({{ $slot }})
    └── Cada módulo se renderiza aquí
```

El layout se encuentra en `resources/views/layouts/app.blade.php`.

Funciona con **Alpine.js** para la interactividad y **Tailwind CSS** para los estilos.

---

### 3.2 Sidebar

- Estado `openSidebar` manejado con Alpine.js (`x-data`)
- En **desktop (>1024px)**: se colapsa a 80px mostrando solo iconos
- En **móvil**: aparece como overlay con fondo negro semitransparente
- Guarda el estado en `localStorage` con clave `sidebar_open`
- Los items de navegación se muestran según permisos del usuario:
  - `dashboard.ver` → Inicio
  - `ventas.ver` → Ventas
  - `pedidos.ver` → Pedidos
  - `clientes.ver` → Clientes
  - `productos.ver` → Productos
   - `usuarios.ver` → Usuarios
   - `reportes.ver` → Reportes (siempre al final del nav, antes de Configuración)
   - `configuracion.ver` → Configuración (abajo del sidebar, fuera del nav)

Cada link usa `request()->routeIs()` para resaltar el módulo activo con clase `bg-[#b9943d]/25 text-[#f2d791]`.

---

### 3.3 Header

- Muestra el `$header` definido en cada vista con `@section('header')`
- Botón hamburguesa visible solo en móvil cuando el sidebar está oculto
- Dropdown de perfil con Alpine.js (`openPerfil`):
  - Nombre y email del usuario
  - Link a Configuración (si tiene permiso)
  - Botón Ayuda → abre modal con número de contacto y WhatsApp
  - Cerrar sesión (form POST a `route('logout')`)

---

### 3.4 Main Content (`{{ $slot }}`)

Cada módulo se envuelve en `<x-app-layout>`:

```blade
<x-app-layout>
    <x-slot name="header">
        <span>Título del módulo</span>
    </x-slot>

    <!-- Contenido del módulo aquí -->
</x-app-layout>
```

El contenido se inyecta en `{{ $slot }}` del layout y el header en `{{ $header }}`.

El main tiene `overflow-y-auto` con `scrollbar-gutter: stable` para scroll independiente sin saltos de layout.

---

### 3.4.1 Modales de formulario

Los formularios de creación con pocos campos (ej. Usuario) se abren en un **modal centrado** en lugar de una ruta separada.

**Estructura del modal:**
- Se activa con Alpine.js (`x-data`, `x-show`, `x-transition`).
- Se renderiza con `x-teleport="body"` fuera del flujo del layout.
- Ancho máximo: `max-w-lg`.
- Overlay semitransparente (`bg-black/60`) que cierra el modal al hacer clic.
- Botón Cerrar en el header: icono `btn-icon-sm` con `cerrar.ico` y fondo `bg-red-600`.
- Botón Cancelar en el footer (texto outline).

**Formulario:**
- POST estándar (no fetch) al `store` del controller.
- Reutiliza el partial `_form.blade.php` con `@include`.
- Botones alineados a la derecha (`justify-end`): Cancelar (outline) → Guardar (sólido `bg-[#111]`).
- Línea divisoria entre el formulario y los botones: `border-t border-gray-200 pt-4` en el contenedor de botones.

**Limpieza de campos:**
Al abrir el modal se limpian todos los campos con JavaScript para evitar datos residuales de `old()`:

```js
abrirNuevo() {
    this.modalNuevo = true;
    setTimeout(() => {
        const f = document.getElementById('nuevo-form');
        if (!f) return;
        f.querySelectorAll('input:not([type=hidden]), select').forEach(el => {
            if (el.tagName === 'SELECT') el.selectedIndex = 0;
            else el.value = '';
        });
    }, 50);
},
```

### 3.5.1 Tamaños de botones

| Clase | Uso | Tamaño | Border-radius |
|-------|-----|--------|---------------|
| `.btn-icon` | Toolbar buttons (Buscar, Filtrar, Roles, Nuevo) | `h-10 w-10` (2.5rem × 2.5rem) | `rounded-xl` (0.75rem) |
| `.btn-icon-sm` | Row action buttons (Editar, Eliminar) | `h-8 w-8` (2rem × 2rem) | `rounded-lg` (0.5rem) |

Ambas clases se definen en un bloque `<style>` dentro del slot de `x-app-layout` e incluyen `display: inline-flex`, `align-items: center`, `justify-content: center`, `transition`, y `flex-shrink: 0`. Los focus/focus-visible tienen `outline: 0 none`.

### 3.9 Flujo de datos

```
Route (web.php)
  → Controller (ej. UsuarioController@index)
    → View (ej. usuarios/index.blade.php)
      → Extiende <x-app-layout> (app.blade.php)
        → Renderiza en {{ $slot }}
```

El middleware `permiso` controla acceso a rutas. El middleware `activo` verifica que el usuario no esté desactivado.

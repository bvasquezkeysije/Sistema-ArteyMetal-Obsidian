### Regla: Filtros en tablas

Todo módulo que muestre una tabla con datos (index) debe incluir un botón o panel de filtros que permita al usuario refinar la búsqueda.

**Campos mínimos sugeridos según el módulo:**

| Módulo | Filtros sugeridos |
|--------|-------------------|
| Usuarios | Rol, Estado (activo/inactivo) |
| Pedidos | Estado, Cliente, Fecha desde/hasta |
| Ventas | Tipo, Estado pago, Fecha desde/hasta |
| Productos | Categoría, Estado (activo/inactivo) |
| Clientes | Documento (DNI/RUC) |
| Reportes | Rango de fechas, Tipo |

**Comportamiento esperado:**
- Los filtros deben persistir en la URL (query string) para poder compartir/enlazar.
- Al aplicar filtros, la tabla debe actualizarse vía request GET al mismo controller.
- Debe haber un botón "Limpiar" o "Quitar filtros" para resetear.
- Los filtros deben colocarse en la misma línea que la barra de búsqueda, o en un panel colapsable debajo de ella.

### Regla: Paginacion

- Texto "Showing X to Y of Z results": `text-sm text-[#8a6a2e]`, numeros `font-semibold text-[#2a2419]`.
- Botones de pagina: `inline-flex h-9 min-w-9 items-center justify-center rounded-lg px-3 text-sm`.
- Pagina inactiva: `border border-[#d1be8a] bg-white text-[#5a4314] hover:bg-[#fff5dd]`.
- Pagina activa: `border border-[#d1be8a] bg-[#f4ebd4] font-semibold text-[#2a2419]`.
- Disabled / ellipsis: `border border-[#e5dec8] bg-[#faf8f2] text-[#8a7a57]`.
- Divisor del contenedor de paginacion: `border-t border-[#efe7d2] px-4 py-3`.

### Regla: Tablas

- La tabla contenedora: `overflow-hidden rounded-2xl border border-[#e5dec8] bg-white shadow-sm`.
- Cabecera (thead): `bg-[#faf8f2] text-left text-[#5a4a2a]`.
- Cuerpo: `divide-y divide-[#efeee9]`.
- Texto de código/id: `font-medium text-[#2d2b24]`.
- Texto de celdas: `text-[#4a4026]`.
- Texto vacío: `text-[#777]`.

### Regla: Panel de filtros (desplegable)

Al presionar el botón Filtrar, el panel de filtros debe desplegarse con `x-show` y `x-transition` (animación suave).

**Comportamiento:**
- El panel se muestra debajo del toolbar, separado por un `border-t border-[#efe7d2] pt-4`.
- Usa `flex flex-wrap items-end gap-4` para que los filtros se alineen horizontalmente y bajen a la siguiente línea si no caben.
- Los selects deben tener fondo blanco (`bg-white`), borde `border-[#d1be8a]`, padding `px-4 py-2.5`, texto `text-sm text-gray-700`, y `shadow-sm`.
- Los labels deben ir arriba de cada select: `mb-1.5 block text-xs font-semibold uppercase tracking-wider text-gray-600`.
- El botón "Filtrar" dentro del panel debe tener el mismo color que el botón de filtrar del toolbar (`bg-sky-500 hover:bg-sky-600`), con `focus:outline-none focus:ring-2 focus:ring-sky-500`.
- Al presionar nuevamente el botón Filtrar del toolbar, el panel se cierra.
- El estado del filtro se refleja en el botón del toolbar con `filter: brightness(0.8)` (clase `.is-active`).

### Regla: Toolbar de tablas (barra de herramientas)

El toolbar de los módulos con tabla debe seguir este orden y estructura:

1. **Barra de búsqueda** (input) → **Botón Buscar** → **Botón Filtrar** → luego los demás botones (Nuevo, etc.)
2. Todos los botones del toolbar deben ser **simétricos**: mismo alto (`h-10`), mismo ancho (`w-10`), misma forma (`rounded-xl`), misma alineación vertical.
3. Los botones de acción (Buscar, Filtrar, Nuevo) deben tener fondo de color sólido con icono blanco. No usar botones outline para acciones principales.
4. Para indicar estado activo (filtro abierto), usar `filter: brightness(0.8)` en vez de rings o borders, para evitar conflictos con el outline nativo del navegador.
5. El card del toolbar es blanco (`bg-white`) con borde `border-[#e5dec8]`. Solo el interior del input de búsqueda lleva fondo `bg-[#fffdf7]` (crema/dorado).
6. El divider del panel de filtros usa `border-[#efe7d2]`.

### Regla: Uso del espacio

- Todo el espacio disponible en la UI debe ser ocupado. No dejar espacios vacíos innecesarios entre elementos.
- Los inputs de búsqueda deben usar `flex-1` para ocupar el ancho disponible.
- Los contenedores deben usar `min-w-0` para evitar desbordamientos en flex containers.

### Regla: Esquema de colores

El sistema usa una paleta con tonos **entre dorado y negro**. Prohibido usar grises puros como fondo de elementos principales.

### Regla: Tamaños de botones

Los botones del sistema usan dos tamaños estándar definidos con clases CSS personalizadas en un bloque `<style>` dentro del slot de `x-app-layout`:

| Clase | Uso | Tamaño | Border-radius |
|-------|-----|--------|---------------|
| `.btn-icon` | Botones del toolbar (Buscar, Filtrar, Roles, Nuevo) | `h-10 w-10` (2.5rem × 2.5rem) | `rounded-xl` |
| `.btn-icon-sm` | Botones de acción en filas (Editar, Eliminar) | `h-8 w-8` (2rem × 2rem) | `rounded-lg` |

**Definición CSS:**

```css
.btn-icon {
    display: inline-flex;
    align-items: center;
    justify-content: center;
    width: 2.5rem;
    height: 2.5rem;
    border-radius: 0.75rem;
    transition: all 0.15s ease;
    flex-shrink: 0;
}
.btn-icon:focus { outline: 0 none; }
.btn-icon:focus-visible { outline: 0 none; }

.btn-icon-sm {
    display: inline-flex;
    align-items: center;
    justify-content: center;
    width: 2rem;
    height: 2rem;
    border-radius: 0.5rem;
    transition: all 0.15s ease;
    flex-shrink: 0;
}
.btn-icon-sm:focus { outline: 0 none; }
.btn-icon-sm:focus-visible { outline: 0 none; }
```

### Regla: Iconos

Todos los iconos del sistema son archivos `.ico` ubicados en `public/icons/`. Se renderizan con `<img>` apuntando a la ruta del archivo.

**Iconos del toolbar:**

| Botón | Archivo | Ruta |
|-------|---------|------|
| Buscar | `buscar.ico` | `/icons/buscar.ico` |
| Filtrar | `filtros.ico` | `/icons/filtros.ico` |
| Roles | `roles.ico` | `/icons/roles.ico` |
| Nuevo (+) | `nuevo.ico` | `/icons/nuevo.ico` |

**Iconos de acciones en filas:**

| Botón | Archivo | Ruta |
|-------|---------|------|
| Editar | `editar.ico` | `/icons/editar.ico` |
| Eliminar / Desactivar | `eliminar-desactivar.ico` | `/icons/eliminar-desactivar.ico` |

**Reglas para iconos:**
- El icono debe tener `width="18" height="18"` para toolbar y `width="16" height="16"` para acciones en fila.
- El icono debe llevar la clase CSS `pointer-events-none` para evitar interferencias con eventos click.
- Los archivos `.ico` se copian desde `E:\SISTEMAS\ICONOS\` a `public/icons/`.

### Regla: Modal de creación (formularios pequeños)

Los formularios de creación con pocos campos (ej. Usuario) deben abrirse en un **modal centrado** encima de la tabla, no en una ruta separada.

**Estructura:**
- El modal se activa con Alpine.js (`x-data`, `x-show`, `x-transition`).
- Se usa `x-teleport="body"` para renderizar el modal fuera del flujo del layout.
- El ancho máximo es `max-w-lg`.
- El fondo tiene overlay negro semitransparente (`bg-black/60`).
- Se cierra al hacer clic en el overlay o en el botón Cerrar/Cancelar.
- El formulario envía vía POST estándar (no fetch) al `store` del controller.

**Limpieza de campos:**
Al abrir el modal, todos los campos deben limpiarse para evitar datos residuales de `old()` (cuando hubo errores de validación en el submit anterior):

```js
abrirNuevo() {
    this.modalNuevo = true;
    setTimeout(() => {
        const f = document.getElementById('nuevo-form');
        if (!f) return;
        f.querySelectorAll('input:not([type=hidden]), select').forEach(el => {
            if (el.tagName === 'SELECT') el.selectedIndex = 0;
            else el.value = '';
        });
    }, 50);
},
```

### Regla: Modales

- Los modales usan borde `border-[#e5dec8]`, header divider `border-[#efe7d2]`, título `text-[#2a2419]`.
- Labels en el modal: `text-xs uppercase tracking-[0.2em] text-[#8a6a2e]`.
- Valores/info en el modal: `text-[#1f1f1f]`.

### Regla: Botones en modales

- Los botones de acción dentro de modales deben alinearse a la **derecha** (`justify-end`).
- El orden estándar es: **Cancelar** (botón outline) → **Acción principal** (botón sólido oscuro `bg-[#111]`).
- Entre el formulario y los botones debe haber una línea divisoria: `border-t border-[#efe7d2] pt-4` en el contenedor de los botones.

### Regla: Botón Cerrar en modales

- El botón **Cerrar** en el header del modal debe ser un icono (`btn-icon-sm`) con el archivo `cerrar.ico` (`/icons/cerrar.ico`), no un botón de texto.
- Fondo: `bg-red-600` (#DC2626), mismo color que Eliminar / Desactivar.
- El icono debe tener `width="16" height="16"` con clase `pointer-events-none`.

### Regla: Indicador de estado activo (filtro abierto)

Cuando el panel de filtros está abierto, el botón Filtrar del toolbar debe reflejarlo visualmente usando `filter: brightness(0.8)` en lugar de rings o borders, para evitar conflictos con el outline nativo del navegador.

```css
.is-active {
    filter: brightness(0.8);
}
```

Se aplica/remueve con Alpine.js:

```blade
<button x-on:click="showFilters = !showFilters"
        :class="{ 'is-active': showFilters }">
```
